## Overview

**AMR-LM** fine-tunes [DNABERT-2](https://github.com/MAGICS-LAB/DNABERT_2) (117M parameters) as a binary classifier for detecting antimicrobial resistance (AMR) genes directly from DNA sequences. The model is trained on the [CARD](https://card.mcmaster.ca/) database and rigorously evaluated for:

- **Cross-database generalization** → MEGARes & SARG test sets
- **Short-read robustness** → 150bp, 300bp, 500bp fragment analysis
- **Data leakage prevention** → k-mer overlap deduplication (k=31, threshold=90%)

The pipeline also benchmarks against alignment-based tools (RGI, DeepARG) and generates publication-ready figures + LaTeX tables.

## Key Results

| Test Set | F1 | MCC | AUROC | Precision | Recall |
|---|---|---|---|---|---|
| CARD (full-length) | **0.954** | 0.919 | 0.984 | 0.933 | 0.977 |
| CARD (500bp) | 0.916 | 0.886 | 0.972 | 0.891 | 0.942 |
| CARD (300bp) | 0.844 | 0.794 | 0.948 | 0.939 | 0.767 |
| MEGARes (full-length) | **0.990** | 0.961 | 0.998 | 1.000 | 0.980 |
| MEGARes (500bp) | 0.952 | 0.875 | 0.977 | 0.984 | 0.923 |

> Cross-database results on MEGARes demonstrate strong generalization (F1=0.99) after rigorous leakage deduplication.

## Pipeline Overview

The end-to-end pipeline consists of 8 sequential steps:

```
1. setup.py                          → Create project directories
2. download_negatives_and_test_dbs.py → Fetch MEGARes, SARG, negative sequences
3. preprocess.py                     → Parse FASTA, deduplicate, create train/dev/test splits
4. run_baselines.py                  → Run RGI & DeepARG baselines
5. train_dnabert2.py                 → Fine-tune DNABERT-2 (full or LoRA)
6. evaluate.py                       → Evaluate on all test sets + fragment lengths
7. generate_paper_figures.py         → Publication-ready figures (300 DPI)
8. generate_paper_tables.py          → LaTeX tables with booktabs
```

## Quick Start

### Prerequisites

- Python 3.8+
- CUDA GPU (recommended; reduces training from ~12h to ~3h)
- CARD database files (downloaded automatically by `setup.py` or manually place in `../card-data/`)

### Installation

```bash
git clone https://github.com/amithgowda-m/AMR-LM.git
cd AMR-LM
pip install -r requirements.txt
```

### Run Full Pipeline

**Windows:**
```batch
run_all.bat
```

**Linux / Mac / WSL:**
```bash
bash run_all.sh
```

**Google Colab:**
Upload `notebooks/amr_pipeline.ipynb` and follow cell-by-cell instructions.

### Run Individual Steps

```bash
python scripts/setup.py
python scripts/download_negatives_and_test_dbs.py
python scripts/preprocess.py
python scripts/run_baselines.py
python scripts/train_dnabert2.py            # add --use_lora for T4 GPUs
python scripts/evaluate.py
python scripts/generate_paper_figures.py
python scripts/generate_paper_tables.py
```

### Training Options

| Flag | Description | Default |
|---|---|---|
| `--use_lora` | LoRA for memory-efficient training | `False` |
| `--epochs N` | Number of training epochs | `5` |
| `--batch_size N` | Training batch size | `8` |
| `--lr RATE` | Learning rate | `3e-5` |
| `--resume` | Resume from best checkpoint | `False` |
| `--max_samples N` | Limit training samples (for quick testing) | `None` |

## Reproducing Results

1. **Clone** this repository
2. **Install** dependencies: `pip install -r requirements.txt`
3. **Run** the full pipeline: `run_all.bat` (Windows) or `bash run_all.sh` (Linux/Mac)
4. All data, models, figures, and metrics will be regenerated in-place

> **Note:** The CARD database (~42 MB) is downloaded automatically by `setup.py`. Model weights (~468 MB) are generated during training and are excluded from the repository.

## Method Summary

1. **Data Collection**: AMR gene sequences from CARD; non-AMR negatives from curated genomic regions; external test sets from MEGARes and SARG
2. **Leakage Prevention**: K-mer overlap analysis (k=31) removes sequences with >90% similarity between training and test databases
3. **Short-Read Simulation**: Full-length sequences are fragmented to 150bp, 300bp, and 500bp to simulate real metagenomic reads
4. **Model**: DNABERT-2 (117M params) with a classification head, trained with class-weighted cross-entropy loss
5. **Evaluation**: Multi-database, multi-fragment-length evaluation with F1, MCC, AUROC, Precision, Recall
